# TokenHarvest Digital Warehouse Receipt (DWR) Module

## **Purpose**

* Digitally represent **stored commodities** as **verifiable, tradable assets**.
* Serve as **collateral for financing** through the Loan Engine.
* Enable **tracking, transfer, and tokenization** of warehouse receipts.
* Ensure **compliance with warehouse receipt regulations** in East Africa.

---

## **1. Actors & Roles**

| Role                     | Responsibility                                           |
| ------------------------ | -------------------------------------------------------- |
| Warehouse Operator       | Issues and updates warehouse receipts                    |
| Borrower / Farmer        | Owns the receipt and can use it for financing or trading |
| Inspection Officer       | Validates commodity before DWR issuance                  |
| Collateral Manager       | Confirms DWR can be used as loan collateral              |
| Lender / Finance Officer | Uses DWR for loan approval                               |
| System / Blockchain      | Ensures immutability and traceability                    |

---

## **2. Workflow Overview**

```text id="dwr_flow"
Commodity Inspected → Inspection Approved → DWR Issued → Tokenized → Collateralized for Loan → Trading / Settlement
```

### Steps:

1. **Link to Commodity**

   * DWR is issued for a **specific commodity batch**.
   * Requires **inspection approval**.

2. **Generate Digital Warehouse Receipt**

   * Receipt contains verified **commodity details, owner, warehouse, and inspection reference**.

3. **Tokenization**

   * Receipt is converted into a **blockchain token** for **finance and trading purposes**.

4. **Integration with Loan Engine**

   * Receipt can be used as **collateral** for borrowing.

5. **Integration with Trading Engine**

   * Receipts can be **listed, traded, or sold** in marketplace.

## **3. Data Captured in DWR**

| Field                | Description                               | Example      |
| -------------------- | ----------------------------------------- | ------------ |
| Receipt ID           | Unique warehouse receipt identifier       | WR-2026-0042 |

| Inspection Report ID | Links to inspection data                  | INS-2026-014 |

| Token ID             | Blockchain token reference                | TH-WR-0042   |
| Status               | Active / Locked / Settled                 | Active       |

## **4. Module Features**

### A. Receipt Issuance

* Automated once inspection is approved.
* Validates **commodity, quantity, and owner**.
* Generates **unique digital ID and QR/barcode** for verification.

### B. Tokenization

* Creates **token on blockchain (IOTA/BNC hybrid)**.
* Enables:

  * Collateralization for loans
  * Transfer to buyers
  * Automated settlement
* Statuses:

  * Active (available)
  * Collateral Locked (used for financing)
  * Settled / Transferred (sold or returned)

### C. Transfer & Trading

* Enables **ownership transfer** without physically moving commodity.
* Integrates with **Marketplace / Futures / Forward Contracts**.
* Can generate **bulk transaction batches** for buyers.

### D. Audit & Reporting

* Every action on DWR is logged in **immutable ledger**: issuance, transfers, collateralization.
* Reports available:

  * Owned receipts per borrower
  * Total commodities in warehouse
  * Receipts locked as collateral

## **5. API Endpoints**

| Endpoint                   | Purpose                                               |
| -------------------------- | ----------------------------------------------------- |
| `POST /dwr/issue`          | Issue a new warehouse receipt                         |
| `GET /dwr/{receipt_id}`    | Retrieve receipt details                              |
| `POST /dwr/tokenize`       | Tokenize receipt on blockchain                        |
| `POST /dwr/transfer`       | Transfer ownership to another user                    |
| `GET /dwr/owner/{user_id}` | List receipts for a specific borrower                 |
| `GET /dwr/status/{status}` | Filter receipts by status (Active / Locked / Settled) |

---

## **6. Security & Validation**

* Only **authorized warehouse operators** can issue DWRs.
* DWR creation requires **inspection approval**.
* Receipt linked to **verified owner ID**.
* Immutable logging ensures **audit trail**.
* Locked tokens prevent **double financing**.
