Below is a **complete financing process flow for the TokenHarvest Loan Engine**, including **approval checkpoints, approver roles, and account creation/management for approvers**. The goal is to ensure **risk control, transparency, and auditable governance** while keeping the process efficient for farmers and lenders.

---

# TokenHarvest Financing Process Flow

## Overview

The financing process follows a **controlled approval workflow** where key stages require **authorized approvers** before funds are disbursed.

```text
Loan Application
      ↓
Collateral Verification
      ↓
Commodity Valuation
      ↓
Credit Risk Assessment
      ↓
Loan Approval
      ↓
Collateral Lock
      ↓
Loan Disbursement
      ↓
Loan Monitoring
      ↓
Repayment / Default Handling
```

Certain stages require **manual approval by designated approvers**.

---

# 1. User Roles in the Financing Workflow

| Role               | Responsibility                                   |
| ------------------ | ------------------------------------------------ |
| Borrower           | Requests financing using warehouse receipt       |
| Collateral Manager | Verifies warehouse receipt and commodity quality |
| Credit Officer     | Reviews loan request and borrower risk           |
| Risk Manager       | Approves high-risk or high-value loans           |
| Finance Officer    | Authorizes loan disbursement                     |
| Platform Admin     | System oversight and role management             |

---

# 2. Stage 1 — Loan Application

### Actor

Borrower (Farmer / Trader / Cooperative)

### Process

Borrower submits a loan request through the mobile or web app.

### Required Data

* Borrower ID
* Warehouse Receipt ID
* Commodity Type
* Quantity
* Requested Loan Amount
* Loan Duration

Example:

```text
Loan Request ID: LN-2026-0012
Borrower: Farmer_0092
Commodity: White Maize
Quantity: 50 Tons
Requested Loan: KES 700,000
Duration: 90 Days
```

### System Action

Loan request created with status:

```text
Status: Pending Collateral Verification
```

---

# 3. Stage 2 — Collateral Verification

### Actor

Collateral Manager

### Responsibilities

The collateral manager verifies:

* warehouse receipt authenticity
* commodity inspection report
* storage location
* commodity quantity and grade

### Approval Decision

```text
Approve Collateral
Reject Collateral
Request Reinspection
```

### System Status Update

If approved:

```text
Status: Collateral Verified
```

---

# 4. Stage 3 — Commodity Valuation

### Actor

System + Risk Engine

### Process

Commodity value calculated using:

* market prices
* historical prices
* AI price prediction

Example:

```text
Commodity: White Maize
Market Price: KES 45,000 per ton
Quantity: 50 Tons
Collateral Value: KES 2,250,000
```

### LTV Calculation

Example rule:

```text
Max LTV: 65%
Max Loan Eligible: KES 1,462,500
```

---

# 5. Stage 4 — Credit Risk Assessment

### Actor

Credit Officer

### Responsibilities

Review:

* borrower credit history
* previous loans
* repayment performance
* collateral value

### Decision Options

```text
Approve Loan
Reject Loan
Request Additional Information
```

### Status Update

```text
Status: Credit Approved
```

---

# 6. Stage 5 — Risk Approval (Large Loans)

### Actor

Risk Manager

This stage applies when:

```text
Loan Amount > Risk Threshold
```

Example threshold:

```text
KES 1,000,000
```

### Responsibilities

* verify risk exposure
* confirm collateral coverage
* approve loan terms

### Decision

```text
Approve
Reject
Reduce Loan Amount
```

---

# 7. Stage 6 — Loan Approval Confirmation

### Actor

Finance Officer

This is the **final approval before funds are released**.

Responsibilities:

* confirm loan conditions
* confirm collateral lock
* authorize disbursement

System update:

```text
Status: Approved for Disbursement
```

---

# 8. Stage 7 — Collateral Lock

### Actor

System (Blockchain Gateway)

Once loan is approved:

```text
Token Status → Locked
```

Example:

```text
Token ID: TH-WR-0042
Lock Type: Loan Collateral
Linked Loan ID: LN-2026-0012
```

The commodity cannot be:

* sold
* transferred
* reused for financing

---

# 9. Stage 8 — Loan Disbursement

### Actor

Finance Officer + System

Supported payment rails:

* M-PESA
* PesaLink
* Paystack
* Stablecoin wallet

Example:

```text
Loan Amount: KES 700,000
Disbursed To: Farmer_0092
Payment Method: M-PESA
Date: 20 July 2026
```

System update:

```text
Status: Active Loan
```

---

# 10. Stage 9 — Loan Monitoring

### Actor

System + Credit Officer

Monitoring includes:

* loan maturity tracking
* commodity price monitoring
* repayment alerts

Notifications sent when:

```text
Loan Due in 7 Days
Loan Overdue
Collateral Value Declining
```

---

# 11. Stage 10 — Repayment

### Actor

Borrower

Repayment channels:

* wallet payment
* M-PESA
* bank transfer
* automatic deduction from commodity sale

Example:

```text
Loan ID: LN-2026-0012
Repayment Amount: KES 700,000
Date: 10 October 2026
```

---

# 12. Stage 11 — Collateral Release

### Actor

System

Once loan balance reaches zero:

```text
Token Status → Active
```

Commodity ownership restored.

---

# 13. Stage 12 — Default Handling

If borrower fails to repay:

```text
Loan Status → Default
```

System triggers liquidation:

* commodity sold in marketplace
* collateral transferred to lender

---

# 14. Approver Account Creation

Approvers must have **special accounts with restricted permissions**.

### Account Roles

| Role               | Permissions               |
| ------------------ | ------------------------- |
| Collateral Manager | Verify warehouse receipts |
| Credit Officer     | Review and approve loans  |
| Risk Manager       | Approve high-risk loans   |
| Finance Officer    | Authorize disbursements   |
| Admin              | Manage system users       |

---

# 15. Approver Account Creation Process

### Admin Creates Account

```text
Admin Dashboard
      ↓
Create Approver Account
      ↓
Assign Role
      ↓
Assign Organization
```

### Required Fields

* Name
* Email
* Phone Number
* Organization
* Role
* Approval Limits

Example:

```text
Name: Jane Mwangi
Role: Credit Officer
Organization: TokenHarvest Finance
Approval Limit: KES 1,000,000
```

---

# 16. Approval Limits

Each approver has a **maximum approval limit**.

Example:

```text
Credit Officer → up to 1M KES
Risk Manager → up to 5M KES
Finance Officer → unlimited
```

If a loan exceeds the limit:

```text
Escalate to next approver
```

---

# 17. Approval Audit Trail

Every approval must be logged.

Example:

```text
Loan ID: LN-2026-0012
Approved By: Credit Officer
Name: Jane Mwangi
Date: 18 July 2026
Decision: Approved
```

This ensures **full traceability**.
