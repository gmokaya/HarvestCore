# TokenHarvest Payment System & Wallet Management Engine

## 1. Purpose

The **Payment & Wallet Engine** manages all financial transactions on the platform.

It enables:

* loan disbursements
* loan repayments
* commodity trade settlements
* wallet balance management
* cross-rail payments (M-PESA, bank, stablecoins)

It also ensures **secure, auditable, and traceable movement of funds**.

---

# 2. Wallet Types

Each user should automatically receive a **multi-currency wallet**.

### Wallet Structure

```text
User Wallet
│
├ Fiat Wallet (KES)
├ Stablecoin Wallet (USDC / USDT)
└ Locked Collateral Wallet
```

### Explanation

**Fiat Wallet (KES)**
Used for:

* loan disbursement
* loan repayment
* commodity purchases
* withdrawal to M-PESA or bank

**Stablecoin Wallet**
Used for:

* lender repayment automation
* cross-border settlement
* DeFi liquidity pools

**Locked Collateral Wallet**
Used for:

* funds reserved for active loans
* escrow in trading contracts

# 5. Supported Transactions

The wallet must support **six key transaction types**.

### 1 Deposit

User deposits money into wallet.

Example:

```text
M-PESA → TokenHarvest Wallet
```

---

### 2 Withdrawal

User withdraws wallet balance.

Example:

```text
Wallet → M-PESA
Wallet → Bank (PesaLink)
```

---

### 3 Loan Disbursement

Loan funds are sent to borrower wallet.

```text
Lender Pool → Borrower Wallet
```

Borrower can withdraw to M-PESA.

---

### 4 Loan Repayment

Borrower repays loan through wallet.

```text
Borrower Wallet → Loan Engine
```

---

### 5 Trade Settlement

Commodity sale proceeds paid to seller wallet.

```text
Buyer Wallet → Seller Wallet
```

---

### 6 Escrow Lock

Funds temporarily locked.

Example:

```text
Forward Contract Settlement
```

---

# 6. Wallet Ledger System

Every wallet transaction must be recorded in a **ledger**.

### Ledger Table

| Field          | Description                         |
| -------------- | ----------------------------------- |
| Transaction ID | Unique transaction identifier       |
| User ID        | Wallet owner                        |
| Amount         | Transaction value                   |
| Currency       | KES / USDC                          |
| Type           | Deposit / Withdrawal / Loan / Trade |
| Status         | Pending / Completed / Failed        |
| Timestamp      | Date and time                       |

Example:

```json
{
 "transaction_id": "TXN-55021",
 "user_id": "FRM-0092",
 "type": "Loan Disbursement",
 "amount": 700000,
 "currency": "KES",
 "status": "Completed"
}
```
