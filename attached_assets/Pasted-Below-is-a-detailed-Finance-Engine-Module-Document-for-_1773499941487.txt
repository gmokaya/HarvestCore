# TokenHarvest Finance Engine

## Detailed Module Document

---

# 1. Purpose of the Finance Engine

The **Finance Engine** is the financial backbone of TokenHarvest. It manages all financial movements within the platform and ensures every transaction is **accurately recorded, auditable, and reconciled**.

### Key Functions

* Wallet balance management
* Loan accounting and repayment tracking
* Escrow and settlement management
* Liquidity pool accounting
* Platform treasury management
* Payment rail reconciliation
* Financial reporting and audit logs

The engine ensures **all financial transactions follow double-entry accounting principles**.

---

# 2. Core Components of the Finance Engine

The Finance Engine consists of several sub-modules:

| Module                    | Function                                                  |
| ------------------------- | --------------------------------------------------------- |
| Wallet Management         | Manages user balances                                     |
| Ledger System             | Records all financial transactions                        |
| Escrow Management         | Holds funds during trades                                 |
| Liquidity Pool Management | Manages lending capital                                   |
| Treasury Management       | Platform revenue and operational funds                    |
| Settlement Engine         | Handles payment processing                                |
| Reconciliation Engine     | Matches platform transactions with external payment rails |

---

# 3. Wallet Management Module

### Purpose

Manages **user wallets and balances** across different currencies.

### Wallet Types

```text
User Wallet
│
├ KES Wallet
├ Stablecoin Wallet
└ Locked Funds Wallet
```

### Wallet Functions

* Create wallet upon user registration
* Deposit funds
* Withdraw funds
* Transfer funds between users
* Lock funds for escrow or loans
* Release locked funds

### Wallet Data Fields

| Field          | Description              |
| -------------- | ------------------------ |
| wallet_id      | Unique wallet identifier |
| user_id        | Wallet owner             |
| currency       | Currency type            |
| balance        | Available balance        |
| locked_balance | Reserved funds           |
| status         | Active / Frozen          |

---

# 4. Financial Ledger System

Every transaction must be recorded in a **double-entry ledger**.

### Ledger Structure

Each transaction creates:

```text
Debit Entry
Credit Entry
```

Example: Loan disbursement

```text
Liquidity Pool (Debit)
Borrower Wallet (Credit)
```

### Ledger Fields

| Field          | Description           |
| -------------- | --------------------- |
| transaction_id | Unique transaction ID |
| debit_account  | Source account        |
| credit_account | Destination account   |
| amount         | Transaction value     |
| currency       | Transaction currency  |
| timestamp      | Transaction date      |

The ledger guarantees **financial accuracy and traceability**.

---

# 5. Escrow Management Module

### Purpose

Temporarily holds funds until a transaction is completed.

### Use Cases

* Commodity trading
* Forward contracts
* Large settlements

### Escrow Workflow

```text
Buyer deposits funds
      ↓
Funds locked in escrow
      ↓
Commodity delivered
      ↓
Funds released to seller
```

### Escrow Data Fields

| Field     | Description                   |
| --------- | ----------------------------- |
| escrow_id | Escrow identifier             |
| buyer_id  | Buyer wallet                  |
| seller_id | Seller wallet                 |
| amount    | Escrow amount                 |
| status    | Locked / Released / Cancelled |

---

# 6. Liquidity Pool Management

Liquidity pools provide **capital for lending and settlements**.

### Pool Types

```text
Liquidity Pools
│
├ Loan Financing Pool
├ Trading Settlement Pool
└ Stablecoin Pool
```

### Pool Operations

* Investor deposits funds into pool
* Loans are funded from pool
* Repayments return to pool
* Returns distributed to lenders

### Pool Data Fields

| Field          | Description                 |
| -------------- | --------------------------- |
| pool_id        | Pool identifier             |
| pool_type      | Loan / Trading / Stablecoin |
| total_balance  | Available funds             |
| locked_balance | Funds allocated to loans    |

# 8. Settlement Engine

The **Settlement Engine** processes payments between wallets and external payment rails.

### Supported Payment Rails

* M-PESA
* PesaLink
* Paystack
* Pesapal
* Stablecoins

### Settlement Workflow

```text
User Wallet
      ↓
Payment Gateway Layer
      ↓
External Payment Rail
```

Example:

```text
Wallet Withdrawal → M-PESA
```

---

# 9. Reconciliation Engine

Ensures the **platform ledger matches external payment systems**.

### Reconciliation Process

```text
Platform Ledger
        ↓
Payment Gateway Records
        ↓
External Payment Rail Logs
```

### Example

```text
M-PESA transaction ID
Wallet ledger transaction
```

Discrepancies trigger alerts.
